# 创建Skill完整指南

## 1. 准备工作
- 确定Skill的功能和目的
- 确认所需的依赖项
- 设计触发条件

## 2. 创建Skill目录结构
```bash
mkdir -p /usr/local/share/nvm/versions/node/v24.11.1/lib/node_modules/openclaw/skills/my-new-skill
```

## 3. 编写SKILL.md
```markdown
# my-new-skill

简短描述Skill的功能。

## 描述
详细描述Skill的作用和功能。

## 触发条件
- 何时激活此Skill
- 用户可能使用的关键词

## 依赖
- 需要安装的工具或库
- API密钥要求

## 示例
- 展示如何使用此Skill
```

## 4. 编写实际功能脚本
- 使用合适的编程语言
- 确保错误处理
- 输出格式化

## 5. 测试Skill
- 在测试环境中验证功能
- 确保所有依赖项正常工作

## 6. 提交到系统
- 将Skill放置在正确目录
- 确保权限设置正确

## 示例：利用新搜索工具的Skill

基于我们新安装的搜索工具，您可以创建一个文件搜索Skill：

1. **创建SKILL.md** - 描述搜索功能
2. **编写脚本** - 使用rg、fd、fzf执行搜索
3. **设置触发条件** - 如"搜索文件"、"查找内容"等关键词
4. **测试功能** - 验证搜索是否正常工作

## 最佳实践
- 保持Skill专注单一功能
- 提供清晰的错误处理
- 记录使用方法
- 考虑性能影响
- 确保安全性
